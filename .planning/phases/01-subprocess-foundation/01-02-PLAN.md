  <name>ffmpeg.py — cross-platform subprocess wrapper</name>
  <files>src/encoder/ffmpeg.py</files>
  <behavior>
    run_ffmpeg(cmd: list[str]):
      - Returns an iterable object (FfmpegProcess) with .cancel() method
      - Iterating yields progress dicts for each "frame=" line in stderr
      - Progress dict keys: frame (int), fps (float), time_seconds (float),
        bitrate (float), speed (float), raw_line (str)
      - Lines that are not progress lines are silently captured (accessible
        via .stderr_lines on the FfmpegProcess object after iteration)
      - After iteration, if ffmpeg exited non-zero, raises FfmpegError
      - Uses subprocess.Popen with CREATE_NEW_PROCESS_GROUP on Windows
      - Uses binary mode stderr + chunk-read + \r-split (never text=True,
        never communicate(), never readline() blocking main thread)
      - Background threading.Thread drains stderr into queue.Queue

    FfmpegProcess.cancel():
      - Writes b"q\n" to proc.stdin, flushes, closes
      - Waits up to 3.0 seconds for process to exit
      - Falls back to proc.terminate() then proc.kill() if needed
      - Does NOT raise FfmpegError after cancel (graceful exit is not an error)
        Implement: set a _cancelled flag; in the post-iteration returncode check,
        skip error raise if _cancelled is True

    FfmpegError:
      - Subclasses Exception
      - __init__(self, returncode: int, stderr: str)
      - self.returncode: int
      - self.stderr: str
      - str(err) includes returncode and first/last lines of stderr

    escape_vmaf_path(path: str | pathlib.Path) -> str:
      - Converts backslashes to forward slashes first
      - On Windows (sys.platform == "win32"): escapes drive colon
        "C:/path" -> "C\\:/path"
        "C:\\path" -> "C\\:/path"
      - On Linux/Mac: returns normalized forward-slash path unchanged
      - Pure function, no subprocess calls
  </behavior>

    Implementation approach:

    1. FfmpegError class first (simplest, needed by everything else):
       ```python
       class FfmpegError(Exception):
           def __init__(self, returncode: int, stderr: str) -> None:
               self.returncode = returncode
               self.stderr = stderr
               super().__init__(
                   f"ffmpeg exited with code {returncode}.\nStderr (last 20 lines):\n"
                   + "\n".join(stderr.splitlines()[-20:])
               )
       ```

    2. escape_vmaf_path (pure function, easiest to verify):
       Convert backslashes → forward slashes, then escape drive colon on Windows.
       See RESEARCH.md Pattern 3 for exact logic.

    3. _parse_progress (internal helper):
       Regex from RESEARCH.md Pattern 4. Returns dict or {"raw_line": line} on no match.

    4. FfmpegProcess class:
       - __init__(self, cmd): stores cmd, sets up internal state
       - __iter__: launches Popen, starts drain thread, yields progress, waits for exit
       - cancel(): implements graceful stop per RESEARCH.md Pattern 2
       - Key fields: _proc (Popen), _cancelled (bool), stderr_lines (list[str])

       The drain thread uses the chunk-read + \r-split approach from RESEARCH.md Pattern 1
       revised drain. Never use proc.stderr iteration (splits on \n only). Never use
       communicate(). Always open stderr in binary mode (no text=True).

       Windows-specific: creationflags=subprocess.CREATE_NEW_PROCESS_GROUP
       Both platforms: stdin=subprocess.PIPE, stdout=subprocess.DEVNULL,
                       stderr=subprocess.PIPE

    5. run_ffmpeg(cmd: list[str]) -> FfmpegProcess:
       Simply returns FfmpegProcess(cmd). This lets callers do:
         gen = run_ffmpeg(cmd); gen.cancel()  # or
         for event in run_ffmpeg(cmd): ...

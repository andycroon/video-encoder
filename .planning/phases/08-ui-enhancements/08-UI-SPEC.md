
VmafChart appears for all jobs that have at least 1 completed chunk (vmaf not null). Visible for both RUNNING and DONE/FAILED jobs. Omits chunks where `vmaf === null` from the data series.

### TopBar controls row (updated for Phase 8)

```
[Encoding Profile selector] ... [Profiles] [Settings] [Theme Toggle] [Add Job]
```

Theme toggle: 38px height, 38px width (square icon button), matching Profiles/Settings button height. Positioned after Settings, before Add Job.

### Theme toggle icon

- Dark mode active: render moon SVG (18×18px, stroke `var(--txt-2)`)
- Light mode active: render sun SVG (18×18px, stroke `var(--txt-2)`)
- No text label — icon only, `title="Toggle theme"` for tooltip accessibility

### Flash prevention

Inline script in `<head>` of `index.html`, executes before React hydration:

```js
(function(){
  var t = localStorage.getItem('theme');
  if (t === 'light') document.documentElement.setAttribute('data-theme', 'light');
})();
```

Default is dark: absence of localStorage key = no `data-theme` attribute = dark mode CSS variables from `:root`.

`useTheme` hook hydrates by reading `document.documentElement.getAttribute('data-theme')` (not localStorage directly) to stay in sync with the already-applied attribute.

### useTheme hook contract

```ts
// Returns
interface UseTheme {
  theme: 'dark' | 'light';
  toggleTheme: () => void;
}
```

- Hydrates `theme` state from `document.documentElement.getAttribute('data-theme') ?? 'dark'` on mount
- `toggleTheme` flips the attribute and writes to `localStorage.setItem('theme', newTheme)`
- Uses `useLayoutEffect` for synchronous DOM read on mount

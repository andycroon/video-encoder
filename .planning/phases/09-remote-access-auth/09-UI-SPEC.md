## Component Inventory

This phase introduces two new full-screen page components and one auth store slice. No existing components are modified.

### New Components

| Component | Description |
|-----------|-------------|
| `LoginPage.tsx` | Full-screen centered card. Username + password fields. Submit button. No nav, no TopBar. |
| `OnboardingWizard.tsx` | Full-screen centered card. Single-step form: username + password + confirm password. Submit button. Progress indicator ("Set up your account"). |
| `ProtectedRoute.tsx` | Wrapper component that checks auth state and renders children or redirects to LoginPage / OnboardingWizard. |

### New Store Slice

| File | Fields |
|------|--------|
| `frontend/src/store/authStore.ts` | `token: string \| null`, `isAuthenticated: boolean`, `setToken(token: string): void`, `clearToken(): void` |

- Use individual Zustand selectors only: `useAuthStore(s => s.token)` вҖ” never inline object selectors.
- JWT persisted to `localStorage` under key `"vce_auth_token"`.

### New API Wrappers

| File | Exports |
|------|---------|
| `frontend/src/api/auth.ts` | `checkAuthStatus(): Promise<{ setup_required: boolean }>`, `login(username, password): Promise<{ access_token: string }>`, `register(username, password): Promise<{ access_token: string }>` |

### Existing API Layer Changes

| File | Change |
|------|--------|
| `frontend/src/api/jobs.ts` (and all api/*.ts) | Add `Authorization: Bearer <token>` header injection. On 401 response: call `clearToken()` from authStore and redirect to login. |

### App.tsx Change

Conditional rendering at root level (no react-router dependency):
1. `GET /api/auth/status` on mount.
2. If `setup_required: true` вҶ’ render `<OnboardingWizard />` full-screen.
3. If `setup_required: false` and `!isAuthenticated` вҶ’ render `<LoginPage />` full-screen.
4. If `isAuthenticated` вҶ’ render existing `<App />` content (TopBar + JobList).

## Auth Routing Logic

```
App mounts
  вҶ“
GET /api/auth/status
  в”ңв”Җ { setup_required: true }  вҶ’ render <OnboardingWizard /> (full-screen, replaces app)
  в”ңв”Җ { setup_required: false } + no token вҶ’ render <LoginPage /> (full-screen, replaces app)
  в””в”Җ { setup_required: false } + valid token вҶ’ render existing app content
```

Loading state during `GET /api/auth/status`:
- Show full-screen `var(--bg)` background only вҖ” no spinner, no skeleton
- Transition is fast (local DB check); flicker prevention not required

401 interception in API layer:
- Clear `localStorage["vce_auth_token"]`
- Call `clearToken()` in authStore
- App re-renders вҶ’ `!isAuthenticated` condition вҶ’ LoginPage appears

## Accessibility

- All inputs have visible `<label>` elements (or `aria-label` if no visible label is used)
- Error messages use `role="alert"` so screen readers announce them immediately
- Submit button `disabled` attribute set (not just `cursor: not-allowed`) when fields are empty or request is in flight
- Password inputs: `type="password"` with `autoComplete="current-password"` (login) / `autoComplete="new-password"` (onboarding)
- Username input: `autoComplete="username"`
- Form submits on Enter keypress in any field (standard `<form onSubmit>` with `type="submit"` button)

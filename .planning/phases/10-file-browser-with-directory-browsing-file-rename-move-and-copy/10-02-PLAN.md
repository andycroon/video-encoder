Create `frontend/src/components/FileBrowser.tsx` — a full-width, full-height dual-panel file browser.

**Component structure:**

```
FileBrowser (top-level)
  +-- FilePanel (left/source)    FilePanel (right/destination)
  Each panel: breadcrumb bar + scrollable file list
  +-- ActionBar (bottom of left panel, visible when selection > 0)
  +-- ConflictDialog (Radix Dialog for overwrite conflicts)
```

**FilePanel sub-component** (render two instances — left and right):
- Props: `path: string`, `onNavigate: (path: string) => void`, `side: 'left' | 'right'`, `selectedPaths: Set<string>`, `onSelectionChange: (paths: Set<string>) => void` (only for left panel)
- State: `entries: BrowseEntry[]`, `parent: string | null`, `currentPath: string`, `loading: boolean`, `error: string`
- On mount and on `path` change, call `browse(path)` and populate entries
- Breadcrumb bar at top showing current path (same style as FilePicker — `var(--bg)` background, `var(--border-lo)` bottom border, monospace font size 12)
- Up navigation: ".." row at top when `parent !== null || currentPath !== ''`
- Directory rows: FolderSvg icon + name (no checkbox per D-10)
- File rows: checkbox (left panel only) + FileSvg icon + name + size + modified_at columns
- Size formatting helper: `formatSize(bytes: number | null)` — returns `''` for null, else `'1.2 GB'`, `'456 MB'`, `'12.3 KB'`, `'890 B'`
- Date formatting: `new Date(modified_at).toLocaleDateString()` or format as `YYYY-MM-DD`

**Selection logic (left panel only, per D-10, D-11, D-12, D-13):**
- Each file row has a checkbox input
- "Select all" checkbox in column header selects/deselects all files in current directory
- `selectedPaths` is a `Set<string>` of file paths
- When `currentPath` changes (navigating away), call `onSelectionChange(new Set())` to clear selection (D-13)

**Action bar (per D-05, D-12):**
- Rendered at bottom of left panel when `selectedPaths.size > 0`
- Shows: `"{N} selected"` text + `[ Move -> ]` button + `[ Copy -> ]` button + `[ x Clear ]` button
- Move button calls `moveFiles(Array.from(selectedPaths), rightPanelPath)` — if any result has `status: 'conflict'`, open ConflictDialog
- Copy button calls `copyFiles(Array.from(selectedPaths), rightPanelPath)` — same conflict handling
- After successful operation (no conflicts or all resolved), clear selection and refresh both panels

**ConflictDialog (per D-17, D-18):**
- Radix `Dialog.Root` — same styling pattern as other modals in the app (`var(--panel)` background, `var(--border)` border, border-radius 8)
- Shows conflicting filename: `"{filename}" already exists at destination.`
- Three buttons: `Overwrite` (calls API again with `overwrite: true` for just that file), `Skip` (removes file from operation list and continues), `Cancel` (aborts entire operation)
- Process conflicts one at a time — show dialog for first conflict, resolve, then check next

**Layout (per D-03):**
- Top-level div: `display: 'flex'`, `gap: 1` (1px divider via border), `height: 'calc(100vh - 80px)'` (below header), `background: 'var(--bg)'`
- Each panel: `flex: 1`, `display: 'flex'`, `flexDirection: 'column'`, `border: '1px solid var(--border)'`, `borderRadius: 8`, `overflow: 'hidden'`, `background: 'var(--panel)'`
- Fixed 50/50 split (Claude's discretion — simpler than draggable divider)

**Panel initial path (per D-06):**
- Both panels start at `''` (root — drives on Windows, / on Linux)
- Path state is component-local (session memory via React state)

**Styling rules:**
- ALL styles use inline style objects with `var(--*)` CSS custom properties
- Reuse FolderSvg and FileSvg from FilePicker.tsx — copy them into this file (they are small SVG components, not worth extracting to shared module)
- Row hover: `onMouseEnter` sets `background: 'rgba(255,255,255,0.03)'`
- Selected row: `background: 'rgba(64,128,255,0.12)'`
- Checkbox: native `<input type="checkbox">` with `accentColor: '#4080ff'`
- Empty state: centered text "No files" in `color: 'var(--txt-3)'` when directory has no video files

**Zustand rules (from MEMORY.md):**
- This component does NOT use Zustand — all state is local (path, entries, selection)
- If it needs to submit a job (Plan 03 adds this), it will use the store with individual selectors only

**TypeScript rules (from MEMORY.md):**
- `import type { BrowseEntry, BrowseResult }` (verbatimModuleSyntax)
- `import type { FileOpResult }` for type-only imports

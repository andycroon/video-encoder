<objective>
Fix CRF oscillation resolution so the pipeline selects the encode whose VMAF score is closest to the center of the target window, not whichever encode the loop happened to land on last.

Purpose: When CRF bounces between two values (e.g. CRF 17 gives VMAF 97.8 and CRF 18 gives VMAF 96.0), the current code returns whichever was last. The fix ensures the mathematically closest-to-center result is selected, with lower CRF as tiebreak.

Output: Modified `_encode_chunk_with_vmaf()` in pipeline.py + 2 new unit tests.
</objective>

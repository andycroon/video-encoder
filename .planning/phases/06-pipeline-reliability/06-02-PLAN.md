<objective>
Add job resume capability so a job that was RUNNING when the application crashed or restarted resumes from its last fully completed pipeline step, skipping already-done work.

Purpose: Without resume, a crash mid-encode forces the entire pipeline to restart from FFV1 encoding. With resume, only the incomplete steps re-run, saving potentially hours of encode time.

Output: Resume gate in run_pipeline, RESUMING status in backend + frontend, PRAGMA busy_timeout in db.py, 3 new tests.
</objective>
